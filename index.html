Introduction to Databases

A database is an organized collection of data, typically stored and accessed electronically from a computer system. Relational databases like MySQL use structured query language (SQL) to manage and manipulate the data.

CREATE Command

The CREATE command is used to create a new database or table in MySQL.

Example:

CREATE DATABASE mydatabase;

CREATE TABLE syntax:

CREATE TABLE customers (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL
);

SELECT Command

The SELECT command is used to retrieve data from one or more tables in MySQL.

Example:

SELECT * FROM customers;

INSERT Command

The INSERT command is used to insert new records into a table in MySQL.

Example:

INSERT INTO customers (name, email) VALUES ('John Doe', 'john@example.com');

UPDATE Command

The UPDATE command is used to modify existing records in a table in MySQL.

Example:

UPDATE customers SET email='john.doe@example.com' WHERE id=1;

DELETE Command

The DELETE command is used to delete existing records from a table in MySQL.

Example:

DELETE FROM customers WHERE id=1;

MySQL Functions

Some commonly used MySQL functions:

  • SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM employees;
  • SELECT UPPER(email) AS uppercase_email FROM users;
  • SELECT LOWER(city) AS lowercase_city FROM addresses;
  • SELECT LENGTH(password) AS password_length FROM users;
  • SELECT AVG(salary) AS average_salary FROM employees;
  • SELECT COUNT(*) AS total_customers FROM customers;
  • SELECT MIN(age) AS min_age FROM users;
  • SELECT MAX(age) AS max_age FROM users;

Constraints

Constraints are used to specify rules for the data in a table. Some commonly used constraints:

  • NOT NULL - Ensures that a column cannot have a NULL value.
  • UNIQUE - Ensures that all values in a column are unique.
  • PRIMARY KEY - Uniquely identifies each record in a table.
  • FOREIGN KEY - Constrains data based on columns in another table.
  • DEFAULT - Provides a default value for a column.
  • CHECK - Ensures that all values in a column satisfy a specific condition.

Joins

Joins are used to combine rows from two or more tables based on a related column between them. Some commonly used joins:

  • INNER JOIN - Returns records that have matching values in both tables.
  • LEFT JOIN - Returns all records from the left table and the matched records from the right table.
  • RIGHT JOIN - Returns all records from the right table and the matched records from the left table.
  • FULL JOIN - Returns all records when there is a match in either left or right table.
  • CROSS JOIN - Returns the Cartesian product of the sets of rows from the joined tables.
